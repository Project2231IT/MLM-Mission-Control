# 2026-04-07 — Daily Memory

## MLM Mission Control — Performance Fix Sprint

### Problem
Dashboard at `reports.misslucillesmarketplace.com` returning "Application error: a client-side exception has occurred" on the Performance page.

### Root Cause
React Error #310 — Rules of Hooks violation. `useMemo` for `rentRows` and `noRentRows` was being called *inside* the `if (!data) return` early-exit block, so hook call order differed between the loading state and the data-loaded state. React 18 is strict about consistent hook call order.

### Fix
Moved all hook calls (useMemo for rentRows, noRentRows, counts, sorted) to always execute regardless of data state. The conditional `if (!data)` only controls the JSX return — hooks run unconditionally first.

Changed:
```tsx
if (!data) return (...);  // early exit — hooks NOT called here
const rentRows = useMemo(...);  // violated Rules of Hooks
```

To:
```tsx
const rentRows = useMemo(() => data ? ... : [], [data]);  // always called
const noRentRows = useMemo(() => data ? ... : [], [data]);
// ... all other hooks
if (!data) return (...);  // conditional render, hooks already called
```

### Files Modified
- `/opt/mlm-mission-control/src/app/performance/page.tsx` — hooks moved before conditional render

# 2026-04-10 — Guest WiFi Captive Portal: WORKING 🎉

## Major Milestone
Guest WiFi captive portal flow is **WORKING END-TO-END** as of ~13:35 CDT.

## The Flow (confirmed working)
1. Guest connects to SSID (e.g. "ACME Guest Wifi")
2. AP redirects to our splash page at `http://172.16.201.40:3700/portal/splash/acme`
3. Guest enters email/phone + name → submits form
4. App saves to PostgreSQL, redirects to `cwp.gwnportal.cloud:8080/gwn_login?username=guest&password=guest`
5. Grandstream cloud sends RADIUS Access-Request to our FreeRADIUS (172.16.201.40:1812)
6. FreeRADIUS accepts (guest/guest in users file)
7. AP grants internet access — guest goes online

## Key Fixes Today
1. **Form submission was empty** — JS used FormData but server expected URL-encoded. Fixed by replacing fetch() with standard form POST
2. **clients.conf too restrictive** — only allowed 172.16.15.0/24. Changed to 0.0.0.0/0 so Grandstream cloud (any IP) can reach FreeRADIUS
3. **Username was MAC address** — Grandstream cloud was rejecting `username=52-46-50-30-AF-6B`. Changed to `username=guest` matching FreeRADIUS users file
4. **Pre-auth rules** — Added `cwp.gwnportal.cloud` to gwn.cloud pre-auth rules so guest devices can reach Grandstream cloud before auth
